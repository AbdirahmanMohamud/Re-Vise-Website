System Architecture

The purpose of the CPU

The microprocessor is the central processing unit(CPU) of the computer. It is here where data processing takes place

The way that the CPU is designed and executes the program instructions is known as ‘von Neumann architecture’.

Von Neumann Architecture

The way that von Neumann proposed the design of the CPU entailed the Control Unit, the Arithmetic and Logic Unit and Registers. These registers are called:

• The accumulator

• The program counter

• The memory address register

• The instruciton register

• The memory data register

The way that the registers work was the program counter would hold a number for the memory address register to go to the address in the RAM. The instruction inside the address would then be loaded to the memory data register which is then copied to the instruction register. That was the fetch stage. In the decode stage, the instruction is followed and then loaded to the accumulator. This was the execute stage. Since the first cycle is complete, then the program counter is updated to 1 for the next cycle.

In the Arithmetic and Logic Unit the following activities are carried out:

• addition and subtraction

• multiplication and division

• logical tests using logic gates

• comparisions, such as whether one number is bigger than another

The Fetch-Decode-Execute Cycle

1. Fetch

The first step the CPU carries out is to fetch some data and instructions (program) from main memory then store them in its own internal temporary memory areas. These memory areas are called 'registers'. This is called the 'fetch' part of the cycle. For this to happen, the CPU makes use of a vital hardware path called the 'address bus'. The CPU places the address of the next item to be fetched on to the address bus. Data from this address then moves from main memory into the CPU by travelling along another hardware path called the 'data bus'. You could imagine that it is a bit like a boat attendant at a lake calling in customers when their time is up -- "Boat number 3, time to come in!" The 'address' of the boat is 3 and the 'data' is its content. The boat is parked at a pier, which is like the internal register of the CPU.

2. Decode

The next step is for the CPU to make sense of the instruction it has just fetched. This process is called 'decode'. The CPU is designed to understand a specific set of commands. These are called the 'instruction set' of the CPU. Each make of CPU has a different instruction set. The CPU decodes the instruction and prepares various areas within the chip in readiness of the next step.

3. Execute

This is the part of the cycle when data processing actually takes place. The instruction is carried out upon the data (executed). The result of this processing is stored in yet another register. Once the execute stage is complete, the CPU sets itself up to begin another cycle once more.

Common Characteristics of CPUs that affect performance

Clock speed – this controls how many calculations the CPU can do in a second so the higher the clock speed, the more efficient it is.

Cache size – the cache is a small amount of primary storage which can allow for far faster processes than going to the RAM with the fetch – decode – execute cycle. Since a larger size of ache would slow the speed, there are levels of cache memory. Level 1 cache is the smallest and closest to the CPU, level 2 cache is larger and further away from the CPU and as a result is slower than Level 1 cache.

Number of cores – this can affect the CPUs performance as the number of cores can multiply the clock speed by 2 or 4 so a CPU with a clock speed of 4.2 Ghz can be slower than a CPU with a clock speed of 2.3Ghz because it has 2 cores or 4 cores.

Embedded Systems

Embedded systems are a computer system built within a larger device. Examples of embedded systems include:

• MP3 Players

• Mobile Phones

• Video Game Consoles

• DVD Players

• Cars

• Dishwashers
